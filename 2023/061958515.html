请简述下Handler线程通信机制

  • Handler负责发送消息sendMessage和处理消息handleMessage消息是封装到Message对象中
  • Message对象存放到消息队列中即MessageQueue
  • Looper负责消息的循环，从消息队列取消息交给对应的Handler来处理。

或者

  • Handler 和 Looper 是 Android 中用于实现线程间通信的关键组件。它们与消息队列（Message Queue）密切相关。
    消息队列（Message Queue）：消息队列是一个先进先出（FIFO）的数据结构，用于存储需要在特定线程中处理的消息（Message）。每个线程都有一个关联的消息队列，用于存储待处理的消息。
  • Looper：Looper 是一个线程本地的消息循环器，通过循环从消息队列中取出消息，并将其分发给相应的 Handler 进行处理。每个线程只能有一个 Looper，并且只能在 Looper 所在的线程中调用 Looper 的相关方法。
  • Handler：Handler 是与特定 Looper 关联的对象，用于发送和处理消息。它可以将消息发送到消息队列中，也可以从消息队列中取出消息并进行处理。Handler 可以关联到特定的线程，并在该线程的消息循环中进行操作。

请简述下Sqlite数据库增删改查的操作步骤

  • 创建SqliteOpenHelper子类，覆写onCreate方法，实现数据库表的创建
  • 通过SqliteOpenHelper对象，调用getWritableDatabase方法获得db对象
  • 通过db相应的insert，delete，update，query方法来实现数据库的增删改查操作，同时过程中会涉及ContentValues数据对象，Cursor对象等。
  • Cursor，db对象使用完毕要及时关闭close，防止内存泄漏。

或者

  • 创建一个继承自 SQLiteOpenHelper 的子类，例如 MyDatabaseHelper。在子类中重写 onCreate() 方法创建数据库表
  • 通过实例化 MyDatabaseHelper 类调用getWritableDatabase方法获得来获取数据库对象。
  • 通过db相应的insert，delete，update，query方法来实现数据库的增删改查操作，同时过程中会涉及ContentValues数据对象，Cursor对象等。
  • Cursor，db对象使用完毕要及时关闭close，防止内存泄漏。

请简述四大组件

  1. Activity (活动).: Activity是Android应用程序的用户界面的基本构建块。它代表一个用户界面屏幕，用户可以与之进行交互。每个Activity都有一个生命周期，可以接收用户输入和处理相关事件。Activity可以启动其他Activity，实现应用程序的导航和交互。

  2. Service(服务） :Service是在后台运行的组件，它可以执行长时间运行的操作而不需要与用户交互。例如，一个Service可以播放音乐、下载文件或执行网络请求。Service通常用于执行耗时的任务，即使用户切换到其他应用程序或者关掉应用程序，Service仍然可以继续运行。

  3. BroadcastReceiver (广摇接收器):BroadcastReceiver是用于接收和响应广播消息的组件。广播消息是一种机制，用于在应用程序之间传递信息。Android系统和应用程序可以发送广播，而BroadcastReceiver可以注册接收并处理这些广播。例如，可以使用广播接收器接收系统的电池低电量警告或接收来自其他应用程序的自定义广播。

  4. ContentProvider(内容提供器): ContentProvider是Android应用程序之间共享数据的一种标准化方式。ContentProvider允许应用程序通过URI(统`资源标识符）访问和共享数据，这些数据可以存储在文件系统、数据库或网络中。ContentProvider可以用于读取和写入数据，它提供了一组标准的接口，其他应用程序可以使用这些接口来访问数据，实现数据共享和交互。
    这四个组件可以相互结合使用，构建出功能强大的Android应用程序。例如，一个应用程序可以使用Activity作为用户界面展示数据，使用Service在后台下载数据，使用BroadcastReceiver接收系统广播消息并做出相应的处理，使用ContentProvider共享数据给其他应用程序使用。

请简述四大启动模式

  • Standard(标准模式)∶这是默认的启动方式。每次启动一个新的Activity时，都会创建一个新的实例，无论该Activity是否已经存在于任务栈中。多次启动同一个Activity会创建多个实例，并按照后进先出的顺序显示在任务栈中。
  • SingleTop(单顶模式):如果要启动的Activity已经位于任务栈的顶部，即它的实例已经存在并位于栈顶，那么不会创建新的实例，而是将现有的实例重用，并调用其onNewIntent0方法来传递新的Intent。如果要启动的Activity不在栈顶，会创建新的实例并将其置于栈顶。
  • SingleTask(单任务模式)︰如果要启动的Activity已经存在于任务栈中(无论是否位于栈顶)，会将其上面的所有Activity全部出栈，使得该Activity成为栈顶，并调用其onNewIntentO)方法。如果要启动的Activity不存在于任务栈中，会创建一个新的实例并将其置于栈顶。
  • Singlelnstance(单实例模式)︰该模式下的Activity会独占一个任务栈，如果要启动的Activity已经存在于任务栈中，会将其上面的所有Activity全部出栈，使得该Activity成为栈顶，并调用其onNewIntent(方法。如果要启动的Activity不存在于任何任务栈中，会创建一个新的任务栈并将该Activity置于栈顶。

启动式服务和绑定式服务的区别

  • 启动式服务（Started Service）：

    • 启动式服务通过调用 startService() 方法启动，并在后台运行独立于调用者的生命周期。
    • 启动式服务不与调用者直接绑定，一旦启动，可以长时间运行，即使启动它的组件被销毁。
    • 启动式服务适用于执行一次性任务、执行后台操作、播放音乐等不需要与调用者进行持续交互的场景。
    • 启动式服务需要在任务完成后调用 stopSelf() 或 stopService() 方法来停止服务。

  • 绑定式服务（Bound Service）：

    • 绑定式服务通过调用 bindService() 方法与调用者绑定，并与调用者共享生命周期。
    • 绑定式服务与调用者建立关联后，可以通过返回的 Binder 对象与调用者进行交互，调用者可以调用服务中的方法。
    • 绑定式服务适用于需要与调用者进行交互、共享数据或提供特定功能接口的场景。
    • 绑定式服务需要在调用者不再需要服务时调用 unbindService() 方法解除绑定。

总结：

启动式服务适用于一次性任务、后台操作、长时间运行的任务，不需要与调用者持续交互。绑定式服务适用于需要与调用者交互、共享数据或提供特定功能接口的场景。

请简述四种常用的存储方法

  • Shared Preferences（共享首选项）：
    • 用于存储轻量级的键值对数据，适用于存储应用程序的配置信息、用户偏好设置等。它是基于 XML 文件存储的，可以通过 SharedPreferences API 进行读写操作。
    • SharedPreferences 是一种用于存储轻量级的键值对数据的存储方法。
    • 使用 SharedPreferences API 可以方便地读写应用程序的配置信息、用户偏好设置等数据。
    • SharedPreferences 存储的数据是基于 XML 文件的，数据会保存在应用程序的私有目录中。
    • 通过调用 getSharedPreferences() 方法获取 SharedPreferences 对象，然后可以使用 putXXX() 方法存储数据，使用 getXXX() 方法获取数据。
  • 文件存储（Internal Storage）：
    • 用于在应用程序的私有目录中存储文件。这些文件仅对应用程序本身可见，其他应用程序无法访问。可以使用 Context 的 openFileOutput() 和 openFileInput() 方法进行文件的写入和读取。
    • 两个类：FileInputStream、FileOutputStream
    • 写有覆盖写和追加写，追加写需要使用模式FileInputSteam(append)
  • SQLite 数据库：
    • 适用于存储结构化数据，提供了一个关系型数据库的解决方案。SQLite 数据库是内置的，使用 SQL 语言进行数据的增删改查操作。
    • SQLite 是 Android 平台内置的关系型数据库管理系统，适用于存储结构化数据。
    • 可以使用 SQLite 数据库来创建和管理数据库，并执行 SQL 语句进行数据的增删改查操作。
    • SQLite 数据库存储的数据是持久化的，即数据会在应用程序关闭后依然保留。
    • 使用 SQLite 数据库需要创建一个继承自 SQLiteOpenHelper 的子类，实现数据库的创建和版本管理。
    • 可以使用 SQLiteDatabase 类提供的方法执行 SQL 语句，如 execSQL()、insert()、update()、delete() 和 query() 等。
  • 网络存储
    通过网络连接将数据存储在远程服务器上或从远程服务器获取数据的过程

ListView和RecyclerView的区别

ListView：

  • Android 中用于显示垂直滚动列表的控件。它可以展示一个垂直方向上的数据集合，并支持用户滚动、选择和交互。

  • 适配器继承自BaseAdapter 或其子类，需要实现关键方法：getCount()、getItem()、getItemId() 和 getView()。

RecyclerView：

  • Android 中用于显示大量数据集合的灵活且高效的控件。它是替代 ListView 和 GridView 的推荐选择

  • 适配器继承自RecyclerView.Adapter，需要实现关键方法：onCreateViewHolder()、onBindViewHolder() 和 getItemCount()。

广播的静态和动态注册， 有序和无序

标准广播（Normal Broadcast）：以异步方式传递给所有匹配的接收者，接收者之间无序接收广播。

有序广播（Ordered Broadcast）：以同步方式传递给所有匹配的接收者，并按照优先级依次接收广播。

静态注册：在 AndroidManifest.xml 文件中声明广播接收者，并指定其接收的广播类型和优先级。

动态注册：通过在代码中调用 registerReceiver() 方法来动态注册广播接收者，并在不需要接收广播时调用 unregisterReceiver() 方法解除注册。
