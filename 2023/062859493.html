测试中的题

生命周期

beforeCreate(创建前）→created(创建后）→beforeMount（载入前)→mounted（载入后）→beforeUpdate（更新前）→updated（更新后→beforeUnmount（销毁前）→unmounted（销毁后)

测试一

理解MVVM模式？

MVVM全称是Model-View-ViewModel，即模型-视图-视图模型。

  1. Model代表整个Web项目所需要的数据模型， Model含有大量信息，但它并不具有任何行为逻辑，它只是数据，因而它不会影响浏览器如何展示数据。
  2. View是视图，具有主动性的，因为它包括了一些数据绑定、事件和行为，这些都会直接影响Model和ViewMode的。
  3. ViewModel：MVVM中的VM它主要负责用一定的业务逻辑对数据进行改变或转换，它也负责将Model的变化反应到View上。

Vue的各种内置指令？

  1. v-model：双向数据绑定;
  2. v-bind：单向数据绑定;
  3. v-on：监听事件;
  4. v-text：插入文本内容;
  5. v-html：插入包含HTML的内容
  6. v-for：列表渲染
  7. v-if：条件渲染
  8. v-show：显示隐藏

computed和监听器的区别？

  1. 计算属性的get必须有return，而监听器可有可无；
  2. 计算属性具有缓存效果，而监听器无；
  3. 计算属性可以自定义名称，而监听器只能监听和data或computed里的名称相同的数据；
  4. 计算属性适用于复杂运算，而监听器适合一些消耗功能，例如Ajax请求

测试二

props的验证方式有哪些？

  1. 基础类型验证：使用type选项指定参数为一种类型
  2. 多种类型验证：type值可以为数组，包含多个数据类型验证
  3. 必需性验证：通过required选项为true来指定props为必需的
  4. 默认值验证：通过default选项为props设置默认值
  5. 自定义验证：通过使用自定义函数进行props验证。自定义函数应该返回true或者false来指示验证结果。

实现JavaScript过渡的钩子函数包括哪些？

  1. beforeEnter：在元素插入前调用
  2. enter：在插入元素的开始调用
  3. afterEnter：在插入元素的结束调用
  4. beforeLeave：在移除元素前调用
  5. Leave：在移除元素的开始调用
  6. afterLeave：在移除元素的结束调用

Transition组件的6个内置的过渡类名？

  1. v-enter-from：进入动画的起始状态
  2. v-enter-active：进入动画的生效状态
  3. v-enter-to：进入动画的结束状态
  4. v-leave-from：离开动画的起始状态
  5. v-leave-active：离开动画的生效状态
  6. v-leave-to：离开动画的结束状态

测试三

动态路由和普通路由有什么区别？

  1. 动态路由和普通路由最大的区别在于是否包含参数。
  2. 普通路由是指固定的路由路径,而动态路由则是包含参数的路由路径。
  3. 在Vue.js中,需要使用不同的方式来定义和获取这两种类型的路由。
  4. 动态路由适用于根据不同的参数或条件加载不同组件的场景，例如博客文章详情页、商品详情页等。而普通路由则适用于那些路径固定不会变化的情况，例如主页、关于页面等。

请简述Vuex 配置对象中的主要内容有哪些。

  1. state:包含应用程序状态的对象。
  2. mutations:包含用于修改 state 的函数。
  3. actions:包含用于触发 mutations 的函数。
  4. getters:包含从 state 中派生出一些状态的函数。
  5. modules:允许将 store 拆分为更小的模块,每个模块都可以具有自己的 state、mutations、actions 和 getters。

要求掌握的题

1.理解MVVM模式？

MVVM全称是Model-View-ViewModel，即模型-视图-视图模型。

  1. Model代表整个Web项目所需要的数据模型， Model含有大量信息，但它并不具有任何行为逻辑，它只是数据，因而它不会影响浏览器如何展示数据。
  2. View是视图，具有主动性的，因为它包括了一些数据绑定、事件和行为，这些都会直接影响Model和ViewMode的。
  3. ViewModel：MVVM中的VM它主要负责用一定的业务逻辑对数据进行改变或转换，它也负责将Model的变化反应到View上。

2.Vue实例配置的各选项

  1. data函数: vue实例数据对象
  2. methods:定义vue实例中的方法
  3. components:定义子组件
  4. computed:计算属性
  5. watch:监听数据变化
  6. filters:过滤器

3.vue的各种内置指令

  1. v-model:双向数据绑定;
  2. v-on:监听事件;
  3. v-bind:单向数据绑定;
  4. v-text:插入文本内容;
  5. v-html:插入包含HTML的内容
  6. v-for:列表渲染
  7. v-if:条件渲染
  8. v-show:显示隐藏

4.v-if和v-show的使用以及区别

①实现原理不同:

  • v-if指令会动态地创建或移除DOM元素，从而控制元素在页面上的显示与隐藏;
  • v-show指令会动态为元素添加或移除style=”display: none;”样式，从而控制元素的显示与隐藏;

②性能消耗不同:

  • v-if有更高的切换开销，而v-show有更高的初始渲染开销
  • 如果需要非常频繁地切换，则使用v-show较好
  • 如果在运行时条件很少改变，则使用v-if较好

5.computed和methods的区别？

  1. 计算属性是依赖的属性发生变化时，才会重新执行代码。methods每次调用都会执行其中的代码。
  2. 计算属性的值会被缓存。methods没有缓存。
  3. 计算属性可以像普通属性一样访问，不需要在后面加括号，methods需要在使用括号进行调用
  4. 计算属性不支持参数传递，methods可以接受参数并进行处理

6.v-on的修饰符有哪些，有何作用？

  1. prevent:阻止默认行为（如阻止超链接的跳转、表单的提交)
  2. stop:阻止事件冒泡
  3. capture:使用事件捕获模式，可以将事件绑定在捕获阶段。
  4. once:绑定的事件只触发1次
  5. self:只有当事件是从触发该监听器的元素自身发出时才触发监听器。

7.props的验证方式有哪些？

  1. 基础类型验证：使用type选项指定参数为一种类型
  2. 多种类型验证：type值可以为数组，包含多个数据类型验证
  3. 必需性验证：通过required选项为true来指定props为必需的
  4. 默认值验证：通过default选项为props设置默认值
  5. 自定义验证：通过使用自定义函数进行props验证。自定义函数应该返回true或者false来指示验证结果。

8.v-model的修饰符有哪些，有何作用？

  1. number:自动将用户的输入值转为数值类型
  2. trim:自动过滤用户输入的首尾空白字符
  3. lazy:在输入框失去焦点或按下回车键时更新绑定的数据，将输入事件触发时立即更新改为延迟更新。

9.transiton组件6个内置的过渡类名以及作用？

  1. v-enter-from：进入动画的起始状态
  2. v-enter-active：进入动画的生效状态
  3. v-enter-to：进入动画的结束状态
  4. v-leave-from：离开动画的起始状态
  5. v-leave-active：离开动画的生效状态
  6. v-leave-to：离开动画的结束状态

10.组件之间实现通信的方式有哪些

父组件向子组件通信：通过 props 和事件来实现父组件向子组件传递数据

子组件向父组件通信：子组件可以通过 this.$emit() 传递数据

创建自定义组件：通过自定义的组件实现通信

使用插槽（Slot）：插槽允许父组件向子组件传递 DOM 结构或其他组件作为子组件的内容。父组件可以在子组件的标记中插入内容，并通过插槽将内容传递给子组件，实现组件之间的通信。

使用事件总线：Vue.js 实例可以作为事件总线，通过创建一个全局的 Vue 实例来充当事件中心，组件可以通过该实例的 $on、$emit 方法来监听和触发事件，实现组件之间的通信。

使用 Vuex 状态管理：Vuex 是 Vue.js 的官方状态管理库，可以用于管理应用程序的状态并实现组件之间的通信。

11.vuex配置选项有哪些

  1. state:包含应用程序状态的对象。
  2. mutations:包含用于修改 state 的函数。
  3. actions:包含用于触发 mutations 的函数。
  4. getters:包含从 state 中派生出一些状态的函数。
  5. modules:允许将 store 拆分为更小的模块,每个模块都可以具有自己的 state、mutations、actions 和 getters。
