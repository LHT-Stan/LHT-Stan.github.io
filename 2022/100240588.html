基础书写和格式化语法

今日目标：

  • 完成Mark Down语法的基本学习
  • 掌握基本的编辑能力
  • 熟练快捷键的使用

标题

要创建标题，# 请在标题前添加一到六个符号，使用的符号的数量#将决定标题的大小。

# 这是一个大型标题

## 这是一个中型标题

###### 这是一个小型标题

当使用两个或者多个标题时，会自动生成一个目录，可以通过文件目录视图来对文章进行访问，每一个标题都列在目录中，单击标题即可导航到所选部分

引用代码

您可以在带有单个反引号的句子中调用代码或命令。反引号内的文本不会被格式化。您还可以按Command+ E(Mac) 或Ctrl+ E(Windows/Linux) 键盘快捷键在 Markdown 行中插入代码块的反引号。

要将代码或文本格式化为自己独特的块，请使用三重反引号。

有关详细信息，请参阅“创建和突出显示代码块”。

如果经常编辑代码片段和表，则可能受益于在 GitHub 上启用对所有注释字段采用固定宽度字体。 有关详细信息，请参阅“在编辑器中启用固定宽度字体”。

支持的颜色模型

在议题、拉取请求和讨论中，您可以使用反引号在句子中标注颜色。反引号内支持的颜色模型将显示颜色的可视化。

以下是当前支持的颜色模型。

颜色 句法 例子 输出
十六进制 #RRGGBB #0969DA
RGB rgb(R,G,B) rgb(9, 105, 218)
HSL hsl(H,S,L) hsl(212, 92%, 45%)

笔记：

  • 支持的颜色模型在反引号内不能有任何前导或尾随空格。
  • 仅在问题、拉取请求和讨论中支持颜色的可视化。

链接

您可以通过将链接文本括在括号[ ]中，然后将 URL 括在括号中来创建内联链接( )。您还可以使用键盘快捷键Command+K创建链接。选择文本后，您可以粘贴剪贴板中的 URL 以自动从所选内容创建链接。

Command您还可以通过突出显示文本并使用键盘快捷键+来创建 Markdown 超链接V。如果您想用链接替换文本，请使用键盘快捷键Command++ 。ShiftV

提示：当评论中写入有效 URL 时，GitHub 会自动创建链接。有关详细信息，请参阅“自动链接的引用和 URL ”。

您可以在渲染的文件中定义相对链接和图像路径，以帮助读者导航到仓库中的其他文件。

相对链接是相对于当前文件的链接。 例如，如果在仓库根目录下有一个自述文件，而在 docs/CONTRIBUTING.md 中有另一个文件，则自述文件中的 CONTRIBUTING.md 的相关链接如下所示 ：

GitHub 将根据您当前使用的分支自动转换相对链接或图像路径，从而使链接或路径始终有效。 链接的路径将相对于当前文件。 以 / 开头的链接将相对于存储库根目录。 可使用所有相对链接操作数，例如 ./ 和 ../。

相对链接更便于用户克隆仓库。 绝对链接可能无法用于仓库的克隆 - 建议使用相对链接引用仓库中的其他文件。

图片

您可以通过!在[ ]. 然后将图片的链接用括号括起来()。

GitHub 支持将图像嵌入到您的问题、拉取请求、讨论、评论和.md文件中。您可以显示存储库中的图像、添加在线图像的链接或上传图像。有关详细信息，请参阅“上传资产”。

提示：当您想要显示存储库中的图像时，您应该使用相对链接而不是绝对链接。

以下是使用相对链接显示图像的一些示例。

语境 相对链接
在.md同一分支的文件中 /assets/images/electrocat.png
在.md另一个分支的文件中 /../main/assets/images/electrocat.png
在问题中，拉取请求和存储库的评论 ../blob/main/assets/images/electrocat.png?raw=true
在.md另一个存储库的文件中 /../../../../github/docs/blob/main/assets/images/electrocat.png
在问题中，拉取请求和另一个存储库的评论 ../../../github/docs/blob/main/assets/images/electrocat.png?raw=true

注意：上表中的最后两个相对链接仅适用于私有存储库中的图像，前提是查看者至少对包含这些图像的私有存储库具有读取权限。

有关详细信息，请参阅“相对链接”。

列表

您可以通过在一行或多行文本前面加上-或来创建一个无序列表*。

要订购您的清单，请在每行前面加上一个数字。

嵌套列表

您可以通过在另一项下方缩进一个或多个列表项来创建嵌套列表。

要使用 GitHub 上的 Web 编辑器或使用等宽字体的文本编辑器（如Atom ）创建嵌套列表，您可以直观地对齐列表。在嵌套列表项的前面键入空格字符，直到列表标记字符 (-或*) 直接位于其上方项目中文本的第一个字符下方。

注意Tab：在基于 Web 的编辑器中，您可以通过首先突出显示所需的行然后分别使用或Shift+来缩进或缩进一行或多行文本Tab。

要在 GitHub 上的评论编辑器中创建不使用等宽字体的嵌套列表，您可以查看嵌套列表正上方的列表项，并计算出现在该项内容之前的字符数。然后在嵌套列表项的前面键入该数量的空格字符。

在此示例中，您可以通过将嵌套列表项100. First list item缩进至少五个空格来在列表项下添加一个嵌套列表项，因为 之前有五个字符 ( 100. ) First list item。

您可以使用相同的方法创建多级嵌套列表。例如，因为第一个嵌套列表项␣␣␣␣␣-␣在嵌套列表内容之前有七个字符 () First nested list item，所以您需要将第二个嵌套列表项缩进七个空格。

有关更多示例，请参阅GitHub 风格的 Markdown 规范。

提到人和团队

您可以在 GitHub 上通过键入加上他们的用户名或团队名称来提及一个人或团队。@这将触发通知并将他们的注意力吸引到对话中。如果您编辑评论以提及他们的用户名或团队名称，人们也会收到通知。有关通知的更多信息，请参阅“关于通知”。

注意：只有当该人对存储库具有读取权限时，才会通知该人提及，并且如果存储库归组织所有，则此人是该组织的成员。

当您提及父团队时，其子团队的成员也会收到通知，从而简化与多组人的沟通。有关详细信息，请参阅“关于团队”。

键入@符号将显示项目中的人员或团队列表。该列表会在您键入时进行过滤，因此一旦您找到要查找的人员或团队的名称，您可以使用箭头键将其选中，然后按 Tab 或 Enter 以完成名称。对于团队，输入@organization/team-name，该团队的所有成员都将订阅对话。

自动完成结果仅限于存储库协作者和线程上的任何其他参与者。

引用问题和拉取请求

您可以通过键入在存储库中调出建议问题和拉取请求的列表#。键入问题或拉取请求编号或标题以过滤列表，然后按 Tab 或 Enter 以完成突出显示的结果。

有关详细信息，请参阅“自动链接的引用和 URL ”。

引用外部资源

如果自定义自动链接引用配置用于仓库，则对外部资源（如 JIRA 议题或 Zendesk 事件单）的引用将转换为缩短的链接。 要了解在您的仓库中哪些自动链接可用，请联系拥有仓库管理员权限的人。 有关详细信息，请参阅“配置自动链接以引用外部资源”。

上传资产

您可以通过拖放、从文件浏览器中选择或粘贴来上传图片等资产。您可以将资产上传到存储库中的议题、拉取请求、评论和.md文件。

使用表情符号

您可以通过键入将表情符号添加到您的写作中:EMOJICODE:。

键入:将显示建议的表情符号列表。该列表将在您键入时进行过滤，因此一旦您找到您要查找的表情符号，请按Tab或Enter以完成突出显示的结果。

有关可用表情符号和代码的完整列表，请查看Emoji-Cheat-Sheet。

段落

您可以通过在文本行之间留空行来创建新段落。

隐藏带有评论的内容

您可以通过将内容放在 HTML 注释中来告诉 GitHub 从呈现的 Markdown 中隐藏内容。

忽略 Markdown 格式

\您可以通过在 Markdown 字符之前使用来告诉 GitHub 忽略（或转义）Markdown 格式。

有关更多信息，请参阅 Daring Fireball 的“ Markdown 语法”。

禁用 Markdown 渲染

查看 Markdown 文件时，可单击文件顶部的 以禁用 Markdown 呈现并改为查看文件的源。

禁用 Markdown 呈现使你能够使用源视图功能，例如行链接，这在查看呈现的 Markdown 文件时不可用。
